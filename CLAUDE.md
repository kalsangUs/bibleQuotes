## Commands

```bash
npm run dev      # Start development server at http://localhost:3000
npm run build    # Production build
npm run start    # Start production server
npx convex dev   # Run Convex backend dev server (regenerates types in convex/_generated/)
```

No test runner or lint script is configured yet.

## Architecture

This is a Next.js 16 app using the App Router (`app/` directory), React 19, Tailwind CSS v4, and Convex as the backend/database.

- **Frontend**: `app/` — Next.js App Router pages and layouts. Entry point is [app/page.tsx](app/page.tsx).
- **Backend**: Convex — serverless functions and database. Add Convex query/mutation/action files under `convex/`. The `convex/_generated/` directory is auto-generated by `npx convex dev` and should not be edited manually.
- **Styling**: Tailwind CSS v4 via PostCSS (`postcss.config.mjs`). Global styles in [app/globals.css](app/globals.css).
- **Path alias**: `@/*` maps to the project root.

### Convex setup

No schema or functions exist yet. To add backend logic:
1. Create `convex/schema.ts` to define tables.
2. Create function files (e.g., `convex/quotes.ts`) exporting `query`, `mutation`, or `action` functions.
3. Run `npx convex dev` to regenerate types.
4. Use `ConvexProvider` and hooks like `useQuery`/`useMutation` from `convex/react` in client components.
